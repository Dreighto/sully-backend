# Operator Visual Review Preferences

Date: 2026-06-12

## Design review rule

The operator does not choose visual directions from descriptions alone. For app icons, logos, UI surfaces, palette choices, and visual identity work, provide a visible batch/contact sheet of options first.

Use plain descriptions only as support after the visual batch exists. The default flow should be:

1. Generate a batch of visual options.
2. Label each option clearly.
3. Let the operator choose by sight.
4. Then refine the chosen direction.

## Current Sully / LogueOS icon context

The operator likes the dot-over-ripple / presence-beacon concept, but it should be translated into the locked Sully / LogueOS indigo visual system unless the operator explicitly asks to revisit magenta.
