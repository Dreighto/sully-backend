Library choice (SvelteKit-compatible)

@lobehub/icons-static RECOMMENDED

Your suggestion. The static-SVG variant of @lobehub/icons — ships every AI brand mark as a plain SVG file, framework-agnostic, no React required. We import the SVGs into $lib/icons/brand/ and load via small Svelte wrappers.

  • Covers every provider you have today + the ones coming (DeepSeek, Grok, GLM, Kimi, etc.)
  • Official brand colors included if we want them tinted; mono fallback works on the chrome
  • One source — no per-provider hand-traced fallbacks (which always look slightly off)
  • MIT-licensed
npm i @lobehub/icons-static-svg

simple-icons

The general-purpose "every brand mark on the internet" package. Has all the AI providers too, but the marks are pure brand glyphs (no per-product variants).

  • Broader catalog (we'd use it if we expand beyond AI — Notion, Linear, GitHub, etc.)
  • Single-color mono SVGs only
  • Slightly more boilerplate to integrate
  • CC0 / public-domain
npm i simple-icons

The pick: ship both — @lobehub/icons-static for the model picker (per-provider brand marks, official colors), simple-icons as a fallback when we add non-AI integrations later (Linear, Notion, etc.). Lucide stays the default for all generic chrome icons (everything that isn't a brand).
